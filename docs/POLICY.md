# Ingestion and Storage Policy

briefcast ingests AI ecosystem updates for personal use via official RSS feeds and public APIs only.

## What we ingest
- Official RSS/Atom feeds from AI labs and research organisations
- Public REST APIs (arXiv export, GitHub Releases, HuggingFace API)
- Public Substack RSS feeds

## What we never do
- HTML scraping of any page
- Bypassing paywalls or login gates
- Polling any source more than once per hour
- Storing full article body text

## What we store
- URL, title, author, source name, publication date
- A 3–5 sentence summary generated by our own LLM pipeline
- A vector embedding of that summary
- Relevance score, deduplication hash, storage mode, soft-delete timestamp

## Removal requests
To request removal of a specific item, open a GitHub issue with the source URL.
All content tables include a `deleted_at` soft-delete column — items are never hard-deleted.
